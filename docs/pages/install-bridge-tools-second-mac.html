Install Bridge Tools on another Mac with SanDisk

Overview

  • Goal: make a second Mac behave like your main Mac for Bridge work:
  • Same SanDisk structure (Bridge_Photos, PhotographyHub, Bridge_Photo_Shortcuts)
  • Same Photography Workflow Hub (cards + Tools row)
  • Same Bridge Tools (Open Bridge (Master), Send Photos to SanDisk, etc.)
  • You can:
  • Use the *same* SanDisk with both Macs (simplest), or
  • Create a *clone* SanDisk that has the same folder layout and hub.

Part A – Make sure the SanDisk is ready

  • On your main Mac (where everything already works):
  • Attach **Extreme SSD**.
  • In Finder, confirm this structure exists:
  • /Volumes/Extreme SSD/Backup_Files_To_NAS/Bridge_Photos
  • /Volumes/Extreme SSD/Backup_Files_To_NAS/PhotographyHub
  • /Volumes/Extreme SSD/Backup_Files_To_NAS/Bridge_Photo_Shortcuts
  • Bridge_Photo_Shortcuts should contain:
  • Bridge_Tools_Launcher.command
  • Open_Bridge_Using_Master.command
  • Open_Bridge_Shot_Dates_Pick_Year.command
  • Build_And_Open_Hub.command
  • Send_Photos_To_SanDisk.command
  • Edit_MASTER_steps.command (or similar)
  • bridge_env.sh

Option 1 – Share one SanDisk between both Macs (recommended)

  • Easiest and safest:
  • Keep using the same **Extreme SSD** on both Macs.
  • PhotographyHub, Bridge_Photos, and Bridge_Photo_Shortcuts live only on this drive.
  • Each Mac:
  • Has its own local TripPhotoBackup folder.
  • Uses the same SanDisk for “master” originals and the hub.
  • Skip to **Part B**.

Option 2 – Clone to a second SanDisk (optional)

  • If you want a separate, identical SanDisk:
  • On your main Mac:
  • Attach **BOTH** drives:
  • Source: current **Extreme SSD** (already in use)
  • Target: new SanDisk (empty or OK to erase)
  • Use a cloning method you trust:
  • Disk Utility “Restore” (drive-to-drive clone), or
  • A backup/clone tool you already use (for example, Carbon Copy Cloner or similar).
  • Important:
  • Give the new drive the same name:
  • **Extreme SSD**
  • So all paths like:
  • /Volumes/Extreme SSD/Backup_Files_To_NAS/...
  • are valid on either drive.
  • After cloning, eject the original and test the new SanDisk by itself:
  • Attach only the new SanDisk.
  • In Finder, confirm:
  • Bridge_Photos
  • PhotographyHub
  • Bridge_Photo_Shortcuts
  • all exist under Backup_Files_To_NAS.

Part B – First-time setup on the second Mac

  • On the new Mac:

Step 1 – Install the apps you need

  • Install **Adobe Bridge 2026** (same major version as the main Mac).
  • Install **Visual Studio Code** (for editing MASTER-steps.txt and scripts).
  • Shortcuts app is built into macOS; no extra install needed.

Step 2 – Attach SanDisk and test basic access

  • Attach **Extreme SSD** to the second Mac.
  • In Finder:
  • Click **Extreme SSD** in the sidebar.
  • Drill down to:
  • Backup_Files_To_NAS → Bridge_Photos
  • Backup_Files_To_NAS → PhotographyHub → index.html
  • Backup_Files_To_NAS → Bridge_Photo_Shortcuts
  • Double-click **Bridge_Tools_Launcher.command**:
  • Confirm the **Photography Workflow Hub** opens in Safari.
  • Check the **Tools** row at the top:
  • Open Bridge (Master)
  • Build & Open Workflow Hub
  • Send Photos to SanDisk
  • Open Bridge (Shot Dates: Pick Year)
  • Edit MASTER-steps.txt

Step 3 – Create the “Bridge Tools” icon on the Desktop

  • In Finder, navigate to:
  • /Volumes/Extreme SSD/Backup_Files_To_NAS/Bridge_Photo_Shortcuts
  • Locate **Bridge_Tools_Launcher.command**.
  • Hold **Option+Command**, then drag **Bridge_Tools_Launcher.command** onto the Desktop:
  • This creates an alias instead of copying the file.
  • Rename the alias to:
  • **Bridge Tools**
  • Optional: change its icon to the Bridge icon (if desired).

Step 4 – Allow .command files to run (first-time only)

  • On the second Mac, open **Terminal** and run:
  • chmod +x "/Volumes/Extreme SSD/Backup_Files_To_NAS/Bridge_Photo_Shortcuts"/*.command
  • If macOS shows security prompts when running a .command:
  • Go to **System Settings → Privacy & Security**.
  • Under “Security”, allow the relevant script if macOS offers an “Open Anyway” button.
  • After approving once, future runs should be smooth.

Part C – Confirm Bridge Tools on the second Mac

1 - Open Bridge (Master)

  • Double-click the **Bridge Tools** icon on the Desktop.
  • On the Workflow Hub Tools row, click **Open Bridge (Master)**.
  • Bridge 2026 should launch and point to:
  • Computer → Extreme SSD → Backup_Files_To_NAS → Bridge_Photos
  • If it points somewhere else:
  • Quit Bridge.
  • Re-run **Open Bridge (Master)**.
  • If it still misbehaves, re-run **Build & Open Workflow Hub** to refresh everything.

2 - Open Bridge (Shot Dates: Pick Year)

  • From **Bridge Tools → Tools row → Open Bridge (Shot Dates: Pick Year)**:
  • You should get a year prompt (e.g., 2025 or 2026).
  • Pick a year:
  • Bridge 2026 should open to:
  • /Users/[your username]/Pictures/TripPhotoBackup/[year]
  • If that folder does not exist yet:
  • The script will create it for you.

3 - Send Photos to SanDisk

  • From **Bridge Tools → Tools row → Send Photos to SanDisk**:
  • A Terminal window should appear and show:
  • “Send Photos to SanDisk starting…”
  • Source: TripPhotoBackup on the second Mac.
  • Destination: Bridge_Photos on the SanDisk.
  • Verify afterwards:
  • New photos appear under:
  • /Volumes/Extreme SSD/Backup_Files_To_NAS/Bridge_Photos/[year]/[shotdate]

4 - Build & Open Workflow Hub

  • From the Tools row, click **Build & Open Workflow Hub**:
  • A Terminal window runs **BuildAllFromMaster.sh**.
  • When it finishes, the Hub reloads in Safari.
  • Confirm:
  • All cards (Importing Photos to Bridge, GPS Tracking, Legacy collections, etc.) appear.
  • The Tools row still shows all five buttons.

5 - Edit MASTER-steps.txt

  • From the Tools row, click **Edit MASTER-steps.txt**:
  • VS Code should open:
  • /Volumes/Extreme SSD/Backup_Files_To_NAS/PhotographyHub/pages/MASTER-steps.txt
  • You can edit and save as usual.
  • Run **Build & Open Workflow Hub** after edits to refresh cards.

Part D – Check bridge_env.sh and user-specific paths

  • On the second Mac, open:
  • /Volumes/Extreme SSD/Backup_Files_To_NAS/Bridge_Photo_Shortcuts/bridge_env.sh
  • In VS Code, confirm:
  • MOUNT points to the right drive:
  • MOUNT="/Volumes/Extreme SSD"
  • SAN_ROOT points at Bridge_Photos:
  • SAN_ROOT="$MOUNT/Backup_Files_To_NAS/Bridge_Photos"
  • TRIP_ROOT uses the current Mac’s username:
  • TRIP_ROOT="$HOME/Pictures/TripPhotoBackup"
  • SEND_SCRIPT (if used) points to the local sender script:
  • SEND_SCRIPT="$HOME/Documents/PhotoBackup/send_photos_to_sandisk.sh"
  • If you ever change folder names on the second Mac:
  • Update bridge_env.sh accordingly.
  • Re-test:
  • Open Bridge (Master)
  • Open Bridge (Shot Dates: Pick Year)
  • Send Photos to SanDisk

Part E – NAS and backups with a second Mac

  • Your NAS and off-site NAS should continue to treat **Extreme SSD** as the main source.
  • If you are using the **same SanDisk**:
  • No change is needed on NAS tasks; you just plug that SanDisk into whichever Mac you are using to manage photos.
  • If you created a **new cloned SanDisk**:
  • Update your NAS backup tasks so they point at the new drive (same path if the volume is named “Extreme SSD”).
  • You do not need to change any Bridge Tools, as long as:
  • The volume name is **Extreme SSD**, and
  • The folder layout under Backup_Files_To_NAS is identical.

Quick sanity checklist for a new Mac

  • Attach **Extreme SSD**.
  • From Finder, double-click the Desktop **Bridge Tools** alias:
  • Photography Workflow Hub appears.
  • Tools row buttons all respond without errors.
  • Test:
  • Open Bridge (Master) → Bridge 2026 opens at Bridge_Photos.
  • Open Bridge (Shot Dates: Pick Year) → prompts for year and opens TripPhotoBackup/[year].
  • Send Photos to SanDisk → new items copied to Bridge_Photos.
  • Edit MASTER-steps.txt → opens in VS Code.
  • Build & Open Workflow Hub → rebuilds cards and reopens the hub.
  • Once this checklist passes, the second Mac is fully ready for Bridge work with your SanDisk.
